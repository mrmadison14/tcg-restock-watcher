## 1.1 [HIGH] Per-store error isolation for event posting
**Files:** `src/tcg_watcher/runner.py` (post-fetch block in `run_once`), `tests/test_runner.py`

- Wrapped `send_events` in a per-store `try/except`; a posting exception is now caught, counted as `stores_failed`, and logged (`[key] post failed: ...`) instead of aborting the whole run.
- `save_snapshot` moved into a `finally` so it always runs once `detect_events` succeeded — encodes the plan's decision: **save the snapshot even when posting fails** (a missed alert beats N duplicates). `stores_ok`/print only happen on the success path.
- TDD: added `test_post_failure_one_store_others_ok` (poster raises for store A → A recorded failed, B processed ok, both snapshots advanced to `last_run="t1"`). Confirmed red before the fix, green after.

## 1.2 [HIGH] O(n²) price-index build
**Files:** `src/tcg_watcher/pricing/build_index.py`, `tests/test_build_index.py`

- Added `_index_prices(price_rows)` → `dict[productId → {"normal", "fallback"}]` built in one pass. `sealed_entries` now builds the index once and `_market_for(pid, index)` is an O(1) lookup (was a full `price_rows` scan per product).
- Subtype-preference semantics preserved exactly: `Normal` `marketPrice` wins; else first non-Normal with a non-null `marketPrice`; `None` prices ignored.
- Equivalence test added (`test_sealed_entries_market_matches_reference_semantics`) covering Normal-preferred, fallback-only, and null-Normal-with-holo-fallback — confirmed green against the original impl first, then unchanged after refactor. `_market_for` has no external callers (grep-verified).

## 1.4 [MED] `_git_show` conflates transient git failure with "file absent"
**Files:** `src/tcg_watcher/reconcile.py`, `tests/test_reconcile.py`

- `_git_show` now: rc 0 → parse JSON; rc 128 **and** stderr contains `"does not exist"` or `"exists on disk, but not in"` → `None` (legit missing path); any other non-zero → `raise RuntimeError` (crashes the reconcile attempt so `commit_state.sh`'s outer retry loop retries).
- Tests added (subprocess mocked via monkeypatch): success/JSON, both missing-path messages → None, transient rc-128 (host unresolved) → raises, rc-1 → raises. `test_commit_state.py` still green.

## 1.5 [LOW] Poster worst-case stall
**Files:** `src/tcg_watcher/http.py` (`make_discord_poster`), `tests/test_discord_poster.py`

- Added `max_429_waits=2` and `total_wait_cap=45.0`. The Discord poster stops retrying after 2 real 429 waits or once the next wait would exceed the 45s cumulative ceiling, then re-raises the last response's `HTTPStatusError`. Generic `make_httpx_get` (GET path) is unchanged.
- Note (from the plan): with 1.1 in place, this raise is now contained per-store.
- Tests: `test_poster_caps_429_waits` (bounded to 2 waits / 3 calls), `test_poster_caps_cumulative_wait` (retry-after 30 → single 30s wait, second would breach 45s → raise). Existing `test_poster_raises_after_exhausting_retries` updated to pass `max_429_waits=10` so it still exercises the `retries`-bounded path (4 calls) as before.
